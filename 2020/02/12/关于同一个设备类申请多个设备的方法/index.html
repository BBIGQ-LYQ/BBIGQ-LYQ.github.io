当一个设备类拥有多个子设备时，如何在一个驱动模块中申请多个设备？.

关于一个设备类下有多个设备，在Linux系统中并不少见

当我们编写的驱动模块需要达到这样的效果时，有些关键的函数和思想不容错过！

关键函数解析.

container_of() 头文件： #include <linux/kernel.h>

根据结构体中的某个成员的地址，从而得到整个结构体的地址

代码原型：

参数 作用
ptr 已知结构体成员的地址
type 要获取整个结构体的类型
member 已知结构体成员的名字


其实就是一个很简单的数学算数问题，我们已知一个 type结构体类型 及type结构体成员名 member 和成员的地址 ptr;

如果我们要计算在结构体type中member的偏移量大小size，我们直接用 size = (&type) - ptr(即 &member);

同理，为了求出type的首地址，我们只需要 &type = ptr - size 即可;

那么关键的 size 怎么求呢？
我们看到了代码原型中有 offsetof 这个函数

这里就涉及到 0指针 的使用了

运行结果：

&((struct type *)0)->k 的意思是：这里的0被强行转化为 struct type * 类型，相当于有一个虚拟的首地址为0的struct type 结构体，并用一个指针指向成员k，而 &((struct type *)0)->k 就是成员k到该结构体首地址的相对距离，也就是求出了结构体成员距离结构体首地址的距离size！

(type *)( (char *)__mptr - offsetof(type,member) )：然后我们只需要用 ptr 已知成员函数的地址减去求出来的 size ，结构体 type 的首地址不就出来了吗！

contianer_of源码中的第一句： const typeof( ((type *)0)->member ) *__mptr = (ptr); ，查阅了一些大佬的博客才突然醒悟，这是内核工程师的一个小手段，就一句话就避免传递的 ptr 与 member 类型不匹配的问题。

源码编写思路.

由于不同的设备需要不同字符设备结构体、设备号以及 对应的设备编号;
那么我们可以定义一个结构体数组，在结构体中包含了字符设备结构体、设备号以及对应的设备编号;

注意事项.

由于不同的次设备使用的都是同一套 file_operations 文件操作集，所以要在对应的文件操作函数中，区分是什么设备调用的，这个时候就涉及到 struct file *file 这个结构体了，通过它进行函数之间的私有数据传递，具体见以下源码！
